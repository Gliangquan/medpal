# Issue: MedPal 多端缺陷整改（第一轮）

## Complexity Assessment

- 等级：L4（系统级）
- 范围：用户端 / 陪诊员端 / 管理员端 / 后端接口 / 管理后台
- 文件规模：预计 10+ 文件
- 风险：跨模块，涉及登录、订单、认证、设置、消息、病历、后台管理
- 不确定性：高，部分功能是缺陷，部分是未实现需求，需分批落地

## Goal

对用户提交的多端问题进行分批修复，优先解决“已有代码但行为错误/缺失交互”的问题，再处理“完全未开发”的模块。

## Problem Buckets

### A. 可快速修复（已有代码基础）
- [ ] 忘记密码入口未开发
- [ ] 预约陪诊时间区间不合理，需按时长自动推导结束时间
- [ ] 预约页取消独立“选科室/选医生”步骤，改为医院内搜索医生/科室
- [ ] 草稿暂存不可用，草稿箱未接入
- [ ] 紧急求助定位获取失败，提交后应立即通知陪诊员
- [ ] 评价后陪诊员星级未同步
- [ ] 设置页清除缓存后直接退登录、隐私设置体验异常
- [ ] 新注册陪诊员未提交资料时认证状态应为“未提交”而不是“审核中”
- [ ] 陪诊员点击完成订单后，不应直接完成，需用户确认

### B. 中等改造（已有页面/接口但数据或权限存在问题）
- [ ] 发布陪诊单时医院/科室/医生信息补全策略

## Risks

- 订单流转变更会影响患者端、陪诊员端、支付后状态展示
- 草稿箱若使用后端草稿接口，需要补充权限校验
- 管理员问题横跨 PC 管理后台与后端权限逻辑，可能超出第一批范围

## Notes

  - 紧急求助页面未保存 latitude / longitude 到 data

  - 订单完成接口允许陪诊员与患者直接完成订单
  - 设置页清缓存后强制 reLaunch 登录页
